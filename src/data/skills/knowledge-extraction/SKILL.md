# 知识萃取 Skill (Knowledge Extraction)

> 知识萃取工具：从课程、文档、视频等素材中提取有价值的知识，沉淀到知识库中

**Skill 版本**：v0.1 (初始框架)
**创建时间**：2026-03-31
**来源**：加号哥需求 - 知识沉淀场景

---

## 📋 Skill 概述

这是一个**知识萃取工具**，用于从各种素材中提取有价值的知识。

**使用方式**：
1. 你把**课程文档、知识笔记、视频脚本、音频转录**等素材发给我
2. 我运用知识框架帮你**萃取核心知识**
3. 我给出**结构化的知识沉淀**
4. 持续完善知识库，让知识可复用

**核心目标**：
- 从海量素材中提取有价值的知识
- 将零散信息转化为结构化知识
- 建立可复用的知识库
- 持续迭代优化知识体系

| 场景 | 说明 |

## 📝 核心概念

### 1. 知识层级

| 层级 | 说明 | 例子 |
|------|------|------|
| **核心概念** | 最基础、最重要的概念和定义 | "什么是 AI Agent" |
| **原则/方法论** | 做事的原则、方法、流程 | "如何设计一个 Skill" |
| **最佳实践** | 经过验证的成功做法 | "Skill 迭代的 5 个步骤" |
| **工具/模板** | 可复用的工具和模板 | "需求澄清 5W2H 模板" |
| **案例/故事** | 实际案例，帮助理解和应用 | "石叔叔案例复盘" |
| **警示/教训** | 失败的教训，避免踩坑 | "不要假设客户的需求" |

### 2. 知识萃取维度

| 维度 | 说明 | 提取重点 |
|------|------|---------|
| **What（是什么）** | 概念、定义、事实 | 核心概念、关键定义 |
| **Why（为什么）** | 原理、原因、动机 | 背后的逻辑、为什么这样做 |
| **How（怎么做）** | 方法、流程、步骤 | 具体操作步骤、方法论 |
| **Who（谁）** | 相关人物、角色 | 专家、案例人物 |
| **When（何时）** | 时间、时机、节奏 | 适用场景、时间节点 |
| **Where（何地）** | 适用场景、环境 | 应用场景、限制条件 |

## 🔧 知识萃取框架

### 阶段一：素材理解
1. 快速浏览素材，了解整体结构
2. 识别素材类型（课程、文档、视频等）
3. 确定素材的核心主题和目标受众

### 阶段二：核心概念提取
1. 提取重要的概念和定义
2. 识别关键的原则和方法论
3. 标记值得注意的警示和教训

### 阶段三：结构化整理
1. 按照知识层级分类整理
2. 建立概念之间的关联
3. 补充必要的背景和上下文

### 阶段四：可复用化
1. 设计可复用的模板（如果适用）
2. 总结最佳实践
3. 提取可操作的行动项

### 阶段五：知识库沉淀
1. 将萃取的知识整理成知识库格式
2. 添加标签和分类，便于检索
3. 记录来源和时间，便于追溯

## 💡 使用 Tips

1. **先理解整体，再提取细节**：不要一开始就逐字逐句，先把握整体结构
2. **关注"为什么"和"怎么做"**：概念重要，但原理和方法更有价值
3. **寻找可复用的模式**：提取模板、方法论、最佳实践
4. **记录来源**：便于后续追溯和深入学习
5. **持续迭代**：知识库不是一次性的，持续补充和优化
6. **用自己的话重述**：用自己的理解重述知识，加深记忆
